Why Analyze Complexity?

To understand resource usage (time and space), compare algorithm efficiency, and predict performance at scale.

  • Big-O (O) — Upper bound: at most this time
  • Big-Omega (Ω) — Lower bound: at least this time
  • Big-Theta (Θ) — Tight bound: exactly this time
لماذا نحلل التعقيد؟

لفهم استخدام الموارد (الوقت والذاكرة)، ومقارنة كفاءة الخوارزميات، والتنبؤ بالأداء.

  • Big-O (O) — حد أعلى: الوقت لا يتجاوز هذا
  • Big-Omega (Ω) — حد أدنى: لا يقل عن هذا
  • Big-Theta (Θ) — حد ضيق: يساوي تماماً

📌 Method 1: Substitution Methodطريقة الاستبدال
How it works

Keep substituting T(n-i) until you reach the base case, then use the Gaussian sum formula.

كيف تعمل

استمر في استبدال T(n-i) حتى تصل للحالة الأساسية، ثم استخدم مجموع Gauss.

Example 1: T(n) = T(n-1) + 1, T(1) = 1 T(n) = T(n-2) + 2 = T(n-3) + 3 = ... = T(n-i) + i Set n-i=1 → i=n-1 → T(n) = T(1) + (n-1) = 1 + n - 1 = n ∈ Θ(n) Example 2: T(n) = T(n-1) + n, T(1) = 1 T(n) = T(1) + 2 + 3 + ... + n = n(n+1)/2 ∈ Θ(n²) ← Gaussian sum: 1+2+...+n = n(n+1)/2 ← مجموع Gauss يجب حفظه!

What is Dynamic Programming?

Solve problems by breaking into overlapping subproblems and storing results (Memoization). Two required properties:

  • Optimal Substructure: Optimal solution built from optimal sub-solutions
  • Overlapping Subproblems: Same subproblem solved multiple times in naive recursion
ما هي البرمجة الديناميكية؟

حل المشاكل بتقسيمها إلى مشاكل فرعية متداخلة وتخزين النتائج (Memoization). شرطان مطلوبان:

  • البنية الفرعية المثلى: الحل الأمثل مبني من حلول فرعية مثلى
  • المشاكل الفرعية المتداخلة: نفس المشكلة الفرعية تُحل مرات متعددة بالعودية الساذجة
🌳 Fibonacci: Overlapping Subproblems Treeفيبوناتشي: شجرة المشاكل المتداخلة
Naive recursion = Θ(2ⁿ) | DP with memoization = Θ(n) — 1000x faster for n=50!
العودية الساذجة = Θ(2ⁿ) | DP = Θ(n) — أسرع بـ1000 مرة عند n=50
Knapsack: I / O / - Symbols
  • I = item IS in the solution (Include)
  • O = item is NOT in solution (Omit)
  • - = no solution found yet (dash)
  • If "-" in last row → NO solution exists for that capacity!

Time & Space: Θ(nK)

Knapsack: رموز I / O / -
  • I = العنصر موجود في الحل
  • O = العنصر ليس في الحل
  • - = لم يُعثر على حل بعد
  • إذا ظهر "-" في الصف الأخير → لا يوجد حل لهذا الحجم!

الوقت والذاكرة: Θ(nK)

اجتياز الجراف: DFS وBFS
🔃 DFS — Depth-First Searchالبحث بالعمق أولاً
How it works
  • Go as deep as possible before backtracking
  • Uses: Stack (or recursion)
  • After visiting v, visit an unvisited neighbor
  • No unvisited neighbor → backtrack
كيف يعمل
  • اذهب بأعمق ما يمكن قبل التراجع
  • يستخدم: Stack (أو العودية)
  • بعد زيارة v، زر جاراً غير مزار
  • لا يوجد جار غير مزار → تراجع
🔄 BFS — Breadth-First Searchالبحث بالعرض أولاً
How it works
  • Visit ALL neighbors before going deeper
  • Uses: Queue (FIFO)
  • Level-by-level exploration
  • Finds shortest path in unweighted graphs!
كيف يعمل
  • زر جميع الجيران قبل التعمق
  • يستخدم: Queue (FIFO)
  • استكشاف مستوى بمستوى
  • يجد أقصر مسار في الجرافات غير الموزونة!
