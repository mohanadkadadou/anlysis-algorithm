📈 Chapter 1: Algorithm Complexity
Big-O Notation · Theta · Omega · Recursive Complexity Analysis
🎯 Why Analyze Complexity?
  • Understand resource usage (time and space)
  • Compare efficiency between recursive vs iterative approaches
  • Predict performance and scalability for large inputs
Big-O (Upper Bound)
T(n) ∈ O(f(n))
Algorithm is at most
f(n) in the worst case

Example: T(n) = 3n² + 5n + 2 → O(n²)

Big-Ω (Lower Bound)
T(n) ∈ Ω(f(n))
Algorithm is at least
f(n) in the best case

Example: Sorting is Ω(n log n)

Big-Θ (Tight Bound)
T(n) ∈ Θ(f(n))
Algorithm is exactly
f(n) (both upper & lower)

Example: MergeSort Θ(n log n)

Growth rates: O(1) ≤ O(log n) ≤ O(n) ≤ O(n log n) ≤ O(n²) ≤ O(2ⁿ)

🔁 Chapter 2: Recurrence Relations
Three Methods: Substitution · Characteristic Equation · Variable Substitution
💡
A recurrence relation expresses T(n) in terms of T for smaller inputs. We solve it to find a closed-form Θ expression.
Method 1: Substitution Method — الطريقة الأبسط

Keep substituting T(n-1), T(n-2), … until you reach the base case, then use the Gaussian sum formula.

📌 Example 1: T(n) = T(n-1) + 1, T(1) = 1
T(n) = T(n-1) + 1 = T(n-2) + 1 + 1 = T(n-2) + 2 = T(n-3) + 3 ... = T(n-i) + i When n - i = 1 → i = n - 1 T(n) = T(1) + (n-1) = 1 + n - 1 = n ∈ Θ(n)
📌 Example 2: T(n) = T(n-1) + n, T(1) = 1
T(n) = T(n-1) + n = T(n-2) + (n-1) + n = T(n-i) + (n-i+1) + ... + n When n - i = 1 → i = n - 1 T(n) = T(1) + 2 + 3 + ... + n = 1 + (n(n+1)/2 - 1) = n(n+1)/2 ∈ Θ(n²)
📐
Gaussian Sum Formula: 1 + 2 + ... + n = n(n+1)/2 — يجب حفظها!
Method 2: Characteristic Equation — الطريقة الرسمية
1
Write in standard form: a_n·t_n + a_{n-1}·t_{n-1} + ... = b₁ⁿ·P₁(n) + b₂ⁿ·P₂(n) + ...

Move all terms to left side. Identify b (base of RHS) and P(n) (polynomial degree d)

2
Build characteristic polynomial:

(homogeneous part) × (x - b₁)^(d₁+1) × (x - b₂)^(d₂+1) × ... = 0

3
Find roots r₁, r₂, r₃, ...

Distinct roots: tₙ = c₁r₁ⁿ + c₂r₂ⁿ + ... | Repeated root r (m times): c₁rⁿ + c₂·n·rⁿ + c₃·n²·rⁿ ...

4
Find constants using initial conditions

Substitute n=1, n=2, ... to build a system of equations and solve for c₁, c₂, ...

RecurrenceRHS: b, dChar. PolynomialRootsGeneral SolutionComplexity
T(n) = T(n-1) + 1b=1, d=0(x-1)(x-1)¹ = 0r₁=r₂=1 (double)c₁ + c₂·nΘ(n)
T(n) = T(n-1) + nb=1, d=1(x-1)(x-1)² = 0r₁=r₂=r₃=1 (triple)c₁ + c₂·n + c₃·n²Θ(n²)
T(n) = 2T(n-1) + 1b=1, d=0(x-2)(x-1)¹ = 0r₁=2, r₂=1c₁·2ⁿ + c₂Θ(2ⁿ)
📌 Full Example: T(n) = T(n-1) + 1, T(1) = 1
Step 1: tₙ - tₙ₋₁ = 1 → b=1, P(n)=1, d=0 Step 2: Char. poly: (x-1) · (x-1)^(0+1) = (x-1)² = 0 Step 3: Root r₁ = r₂ = 1 (double root) General solution: tₙ = c₁·1ⁿ + c₂·n·1ⁿ = c₁ + c₂·n Step 4: t₁ = 1 = c₁ + c₂·1 t₂ = t₁+1 = 2 = c₁ + c₂·2 Solving: c₁ = 0, c₂ = 1 Result: tₙ = n ∈ Θ(n) ✓
Method 3: Variable Substitution — for T(n/b) style

When recurrence uses division (n/2), substitute n = 2^k to convert to a simpler form.

📌 Example: T(n) = T(n/2) + 1, T(1) = 1
Let n = 2^k → k = log₂n T(2^k) = T(2^(k-1)) + 1 → t_k = t_{k-1} + 1 Now solve: t_k - t_{k-1} = 1 → b=1, d=0 Char. poly: (x-1)² = 0 → r₁=r₂=1 t_k = c₁ + c₂·k Substituting back: T(n) = c₁ + c₂·log₂n Result: T(n) ∈ Θ(log n)
📌 Example: T(n) = 2T(n-2) + 1, T(1) = T(2) = 1
tₙ - 2tₙ₋₂ = 1 → b=1, d=0 Char. poly: (x² - 2)(x - 1)^(0+1) = 0 Roots: r₁ = √2, r₂ = -√2, r₃ = 1 General: tₙ = c₁·(√2)ⁿ + c₂·(-√2)ⁿ + c₃·1ⁿ Result: T(n) ∈ Θ(√2ⁿ)
🧮 Chapter 3: Master Theorem
Quick solver for divide-and-conquer recurrences T(n) = aT(n/b) + f(n)
📐 Master Theorem Formula
Given: T(n) = a·T(n/b) + f(n) where f(n) ∈ Θ(nᵈ), d ≥ 0 Case 1: a < bᵈ → T(n) ∈ Θ(nᵈ) Case 2: a = bᵈ → T(n) ∈ Θ(nᵈ log n) Case 3: a > bᵈ → T(n) ∈ Θ(n^(log_b a)) Note: Same results hold with O instead of Θ
Case 1: a < bᵈ

The non-recursive part dominates.

T(n) ∈ Θ(nᵈ)

Example: T(n) = 4T(n/2) + n³
a=4, b=2, d=3 → bᵈ=8 > a=4 ✓

T(n) ∈ Θ(n³)
Case 2: a = bᵈ

Both parts equally contribute.

T(n) ∈ Θ(nᵈ log n)

Example: T(n) = 4T(n/2) + n²
a=4, b=2, d=2 → bᵈ=4 = a=4 ✓

T(n) ∈ Θ(n² log n)
Case 3: a > bᵈ

The recursive part dominates.

T(n) ∈ Θ(n^(log_b a))

Example: T(n) = 4T(n/2) + n
a=4, b=2, d=1 → bᵈ=2 < a=4 ✓

T(n) ∈ Θ(n²)

💡 Chapter 4: Dynamic Programming
Memoization · Fibonacci · Binomial Coefficient · Knapsack
🧠 What is Dynamic Programming?
  • Solve problems by breaking into overlapping subproblems
  • Store solutions to avoid solving the same subproblem twice → Memoization
  • Two required properties: Optimal Substructure + Overlapping Subproblems
⚡
Recursion + Memoization = Dynamic Programming. Difference: in pure recursion, same subproblems solved multiple times!
🌳 Fibonacci: Overlapping Subproblems Visualized
Naive recursion recomputes fib(3) and fib(2) multiple times → Θ(2ⁿ). DP stores results → Θ(n)
ApproachTimeSpaceKey Feature
Naive RecursionΘ(2ⁿ)Θ(n) stackRecomputes same values exponentially
DP / MemoizationΘ(n)Θ(n) tableEach fib(i) computed exactly once
📊 Binomial Coefficient C(n,k) — DP Table
C(n, k) = 1 if k=0 or k=n C(n, k) = C(n-1, k-1) + C(n-1, k) if 0 < k < n Time: Θ(nk) Space: Θ(nk)
Pascal's Triangle as DP table. Each cell = sum of two cells above it.
🎒 Knapsack Problem — Key Concepts

Problem: Given integer K and n items with sizes k₁,k₂,...,kₙ, find a subset summing to exactly K (or determine none exists).

SymbolMeaning
ISolution containing the item (item IS in the subset)
OSolution without the item (item is NOT in subset)
-No solution found yet for this subproblem
"-" in last rowNO solution exists for knapsack of that size!
DP State: P[i,k].exist = true if first i items can fill knapsack of size k P[i,k].exist = true if: (1) P[i-1, k].exist is true → don't include item i OR (2) k ≥ S[i] AND P[i-1, k-S[i]].exist is true → include item i Example trace from slides (items: k1=3, k2=5, k3=6, k4=4, K=16): P(4,16) = I → include item 4 (size=4) P(3,16-4=12): but slide shows P(3,16-6)=I → include item 3 (size=6) P(2,10-5)=I → include item 2 (size=5) P(1,5-3)=I → include item 1 (size=3) Total: 3+5+6+4 = 18? ... Actually 3+5+6 = 14 ≠ 16. Slide shows sizes sum to K. Time: Θ(nK) Space: Θ(nK)

📐 Degrees in Graphs

Undirected Graph: degree(v) = number of edges touching v

Sum of all degrees = 2|E| (handshaking lemma)

d⁺(v): arrows leaving v

🔍 Chapter 6: DFS & BFS
Depth-First Search · Breadth-First Search · Queue/Stack Traces
🔃 DFS — Depth-First Search
  • Go as deep as possible before backing up
  • After visiting v, visit an unvisited adjacent vertex
  • If no unvisited adjacent → backtrack
  • Uses: Stack (implicit in recursion)
void dfs(v: Vertex) { mark v as visited; for (each unvisited u adjacent to v) dfs(u); }
🔄 BFS — Breadth-First Search
  • Visit ALL neighbors of current vertex first
  • Then move to next level
  • Level-by-level exploration
  • Uses: Queue (FIFO)
void bfs(v: Vertex) { queue.enqueue(v); mark v; while (!queue.isEmpty()) { w = queue.dequeue(); for (each unvisited u adjacent to w) { mark u; queue.enqueue(u); } } }

🧩 Chapter 7: Algorithm Design by Induction
N! · xN · Polynomial Evaluation · Maximal Induced Subgraph · One-to-One Mapping
📐 Mathematical Induction — 3 Steps
1
Induction Hypothesis: Assume we know how to solve the problem for size N-1
2
Base Case: Prove the statement is true for the smallest value (n=1)
3
Inductive Step: Use the N-1 solution to construct the N solution
💡
Gaussian Sum Proof: 1+2+…+(n-1) = (n-1)n/2 → add n → n(n+1)/2 ✓
Example 1: N! (Factorial)
Assume we know (N-1)! N! = (N-1)! × N T(n) = T(n-1) + O(1) T(n) ∈ O(n)
int factorial(int N) { if (N == 1) return 1; return N * factorial(N-1); }
Example 2: xN (Power)
Assume we know x^(N-1) x^N = x^(N-1) × x T(n) = T(n-1) + Θ(1) T(n) ∈ Θ(n)
float power(float x, int N) { if (N == 0) return 1; return x * power(x, N-1); }
🌐 Maximal Induced Subgraph — Key Algorithm

Problem: Given undirected graph G=(V,E) and integer k, find the largest induced subgraph where ALL vertices have degree ≥ k.

1
Check all degrees: If all vertices have degree ≥ k → the whole graph is the answer!
2
Find a vertex v with degree < k: This vertex CANNOT be in any valid subgraph (its degree will always be < k)
3
Remove v and all its edges: Apply the algorithm to the remaining n-1 vertices
4
Repeat until no more vertices can be removed
Each vertex in final subgraph has degree ≥ k. Removals are mandatory!
⭐
Dr.'s Exam Q5: k=2, vertices {1,2,3,4,5,6,7}, edges: (1,2),(1,3),(2,3),(3,4),(4,5),(4,6),(5,6),(6,7). Answer: {4,5,6} — trace by removing vertices with degree < 2!
⚡ Chapter 8: Divide & Conquer
Binary Search · MergeSort · Karatsuba · Strassen
🔑 D&C General Strategy
D
Divide: Split the problem into 2+ smaller subproblems of the same type
C
Conquer: Solve each subproblem recursively (base case: solve directly)
M
Merge/Combine: Combine solutions to get the answer to original problem
General recurrence: T(n) = a·T(n/b) + f(n)
a = number of subproblems, b = size shrink factor, f(n) = combine cost
🔎 Binary Search
T(n) = T(n/2) + O(1) By Master Theorem: a=1, b=2, d=0 b^d = 1 = a → Case 2 T(n) ∈ Θ(log n)

🔢 Large Integer Multiplication & Strassen

Grade-school: n² single-digit multiplications → Θ(n²)

Naive D&C: 4 multiplications → T(n) = 4T(n/2) + cn → Θ(n²)

Karatsuba: Use 3 instead of 4! A₁×B₂ + A₂×B₁ = (A₁+A₂)(B₁+B₂) - A₁B₁ - A₂B₂ T(n) = 3T(n/2) + O(n) T(n) ∈ Θ(n^log₂3) ≈ Θ(n^1.585)

Strassen: 7 products instead of 8 for matrix multiplication

M₁ = (A₁₁+A₂₂)(B₁₁+B₂₂) M₂ = (A₂₁+A₂₂)B₁₁ ... (7 products total) T(n) = 7T(n/2) + O(n²) T(n) ∈ Θ(n^log₂7) ≈ Θ(n^2.807) vs naive O(n³)

Q2HARDRecurrence

Q3HARDChar. Equation
For T(n) = T(n-1) + 1, using characteristic equation method: tₙ - tₙ₋₁ = 1 gives roots r₁=r₂=1. The general solution is:
Double root r=1: (x-1)(x-1)¹=0 → r₁=r₂=1. With repeated root, solution is c₁·1ⁿ + c₂·n·1ⁿ = c₁ + c₂·n. Apply T(1)=1,T(2)=2 → c₁=0, c₂=1 → tₙ=n ∈ Θ(n).
Q4HARDChar. Equation
For T(n) = T(n-1) + n, tₙ - tₙ₋₁ = n. Here b=1, P(n)=n (degree d=1). The characteristic polynomial is:
Build char. poly: (x-1) from homogeneous × (x-b)^(d+1) = (x-1)^(1+1) = (x-1)². Total: (x-1)·(x-1)² = (x-1)³ = 0. Triple root → general solution c₁+c₂n+c₃n² ∈ Θ(n²).
Q5HARDChar. Equation
For T(n) = 2T(n-1) + 1, applying char. equation gives roots r₁=2, r₂=1. General solution is tₙ = c₁·2ⁿ + c₂. With T(1)=1, T(2)=3, what is c₁?
Apply initial conditions: t₁=1: 2c₁+c₂=1. t₂=3: 4c₁+c₂=3. Subtracting: 2c₁=2 → c₁=1. Then c₂=1-2=-1. So tₙ=2ⁿ-1 ∈ Θ(2ⁿ).
Q6MEDMaster Theorem
Apply Master Theorem to T(n) = 4T(n/2) + n. a=4, b=2, d=1. bᵈ=?
bᵈ = 2¹ = 2. Compare: a=4 > bᵈ=2 → Case 3. T(n) ∈ Θ(n^log₂4) = Θ(n²).
Q7MEDMaster Theorem

Strassen's algorithm has T(n) = 7T(n/2) + O(n²). What is log₂7 approximately?
log₂7 = ln7/ln2 ≈ 1.946/0.693 ≈ 2.807. So Strassen ∈ Θ(n^2.807), better than naive O(n³) matrix multiplication.

Q12MEDBig-O
What is the time complexity of naive Fibonacci recursion fib(n) = fib(n-1) + fib(n-2)?
Exponential growth: Each call spawns 2 calls. The call tree has ~2ⁿ nodes. With memoization (DP), it drops to Θ(n).

Q83HARDD&C - Binary Search
Binary search RECURSIVE version has space complexity O(log n) while ITERATIVE version has O(1). Why?
Call stack depth: Each recursive call creates a new stack frame. Binary search halves the array each time, so maximum depth is log₂n. Iterative version just updates variables in a loop → O(1) extra space.
Q84HARDD&C
In MergeSort, the 'merge' operation on two sorted arrays of sizes m and n takes:
Linear merge: We scan both arrays once, comparing elements and copying. Total operations = m+n elements to copy → Θ(m+n). This is why the combine step in MergeSort recurrence is O(n).
Q85HARDInduction
Algorithm design by induction: FIRST step is to assume we know the solution for N-1. SECOND step is to use that solution for N. Why is this valid?
Induction validity: By assuming we can solve N-1, we only need to show how to EXTEND this to N. This works because N-1 < N (strictly smaller), so eventually we reach the base case. The key is that the N-1 subproblem is EXACTLY the same problem type, just smaller.

What is the key distinction between Divide and Conquer vs Decrease and Conquer?
From slides: 'Distinction: Decrease and Conquer typically does not solve both subproblems.' D&C splits into 2+ equal parts and solves ALL. Dec&C reduces problem size by 1 (or constant factor) and solves the ONE reduced problem.
